# Problem Solving Patterns (Engineering Thinking)

## How to Approach DSA Problems

Solving problems isn't about memorizing solutions — it's about **recognizing patterns**.

## Step-by-Step Framework

### 1. Understand the Problem
- Read carefully, identify inputs and outputs
- Work through examples by hand
- Ask: what are the constraints?

### 2. Identify the Pattern
| If you see... | Think about... |
|---------------|---------------|
| Sorted data | Binary search |
| Finding pairs | Two pointers or hash map |
| Contiguous subarray | Sliding window |
| All possibilities | Backtracking |
| Optimal solution from subproblems | Dynamic programming |
| Locally optimal → global | Greedy |
| Level-by-level | BFS (queue) |
| Go deep first | DFS (stack/recursion) |

### 3. Start with Brute Force
Always get a working solution first, then optimize.

### 4. Optimize
- Can I reduce time complexity?
- Can I trade space for time?
- Am I doing redundant work?

## Optimization Mindset

```
Brute Force → Identify bottleneck → Apply pattern → Optimize
```

## Key Points

- Start with brute force — correctness first
- Recognize which pattern applies
- Optimize step by step
- Know the common trade-offs: time vs space
- Practice builds pattern recognition
